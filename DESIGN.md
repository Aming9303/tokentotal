# TokenTotal 设计稿

## 定位

TokenTotal 是一个面向 AI 应用、客服知识库、内容工作流和批量调用场景的 token 用量量化工具。核心目标不是只显示一个 token 数，而是把“输入规模、输出预算、上下文风险、缓存收益、模型差价、月度调用成本”放到同一个工作台里。

## 目标用户

- 产品经理：快速判断某个功能的 token 成本区间。
- 开发者：在 prompt、RAG 片段、聊天记录上线前做容量评估。
- 运营/财务：按日调用量估算月度 API 成本。
- 方案顾问：比较不同模型组合的成本和上下文余量。

## MVP 信息架构

1. 输入区：文本粘贴、文件导入、清空、样例。
2. 预算参数：模型、输出 token、上下文上限、缓存输入比例、每日调用、工作天/月。
3. 核心指标：输入 token、单次成本、月成本、上下文占比。
4. 文本拆解：字符、英文词、CJK 字符、行数、消息数、代码密度、token 来源分布。
5. 模型对比：同一输入下的上下文预算、单次成本和月成本。

## 计算模型

当前版本使用浏览器端启发式估算：

- CJK 字符按接近单字符 token 估算。
- 英文按平均 4 个字符一个 token 估算。
- 数字、符号、空白、代码密度做额外修正。
- 聊天记录可叠加每条消息协议开销。

生产版建议升级为三层精度：

- 快速估算：纯前端，无依赖，适合方案阶段。
- 精确本地：后端接入 `tiktoken`，返回模型对应 token 数。
- 账单校准：导入真实 API usage，把估算误差反向校准到团队模板。

## 后续扩展

- 自定义模型价格和上下文窗口，并导入/导出配置。
- Prompt 模板变量批量展开，计算 P50/P90/P99 token。
- RAG 场景拆分 system、query、retrieved chunks、tools、output。
- 多模型路由成本模拟，例如 cheap model 初筛、strong model 兜底。
- 接入真实 API usage CSV，生成部门、项目和功能维度的成本报表。
